# FMEA Input Schema
**Project:** FMEA Risk Prioritization Tool
**Version:** 1.0 | March 26, 2026

This document defines the required format for FMEA input files (CSV or Excel).

---

## Required Columns

| Column Name | Data Type | Valid Range | Description |
|-------------|-----------|-------------|-------------|
| `ID` | Integer | ≥ 1 | Unique row identifier |
| `Process_Step` | String | Any | Manufacturing process step (e.g., "Autoclave Cure") |
| `Component` | String | Any | Part or assembly name (e.g., "Vacuum Bag") |
| `Function` | String | Any | What the component/step is supposed to do |
| `Failure_Mode` | String | Any | How it can fail (e.g., "Bag rupture during cure") |
| `Effect` | String | Any | Consequence of the failure mode |
| `Severity` | Integer | 1–10 | How severe is the effect? (10 = safety hazard) |
| `Cause` | String | Any | Root cause of the failure mode |
| `Occurrence` | Integer | 1–10 | How often does this cause occur? (10 = almost certain) |
| `Current_Control` | String | Any | Existing detection or prevention control |
| `Detection` | Integer | 1–10 | How well can current controls detect it? (10 = no detection) |

## Calculated Columns (auto-generated by tool)

| Column Name | Formula | Description |
|-------------|---------|-------------|
| `RPN` | Severity × Occurrence × Detection | Risk Priority Number |
| `Flag_High_RPN` | RPN > 100 | True if RPN exceeds action threshold |
| `Flag_High_Severity` | Severity ≥ 9 | True if safety-critical severity |
| `Flag_Action_Priority_H` | RPN ≥ 200 OR Severity ≥ 9 | True if Action Priority = High |
| `Risk_Tier` | See ASSUMPTIONS_LOG Rule 4 | Red / Yellow / Green |

## Scoring Scales (AIAG FMEA-4)

### Severity (S)
| Score | Meaning |
|-------|---------|
| 10 | Safety hazard — no warning |
| 9 | Safety hazard — with warning |
| 8 | Loss of primary function |
| 7 | Reduced primary function |
| 6 | Loss of secondary function |
| 5 | Reduced secondary function |
| 4 | Cosmetic defect — noticed by most customers |
| 3 | Cosmetic defect — noticed by some customers |
| 2 | Minor defect — noticed by discriminating customers |
| 1 | No effect |

### Occurrence (O)
| Score | Meaning | Approximate Rate |
|-------|---------|-----------------|
| 10 | Almost certain | ≥ 1 in 2 |
| 9 | Very high | 1 in 8 |
| 8 | High | 1 in 20 |
| 7 | Moderately high | 1 in 80 |
| 6 | Moderate | 1 in 400 |
| 5 | Low | 1 in 2,000 |
| 4 | Slight | 1 in 15,000 |
| 3 | Very slight | 1 in 150,000 |
| 2 | Remote | 1 in 1,500,000 |
| 1 | Almost impossible | < 1 in 1,500,000 |

### Detection (D)
| Score | Meaning |
|-------|---------|
| 10 | No detection control exists |
| 9 | Very remote chance of detection |
| 8 | Remote chance of detection |
| 7 | Very low chance of detection |
| 6 | Low chance of detection |
| 5 | Moderate chance of detection |
| 4 | Moderately high chance of detection |
| 3 | High chance of detection |
| 2 | Very high chance of detection |
| 1 | Almost certain detection |

## Sample Row

```csv
ID,Process_Step,Component,Function,Failure_Mode,Effect,Severity,Cause,Occurrence,Current_Control,Detection
1,Autoclave Cure,Vacuum Bag,Maintain vacuum integrity during cure,Bag rupture during cure,Part exposed to atmosphere — resin bleed-out — void formation,9,Bag puncture from sharp prepreg edges,3,Visual inspection before cure,4
```
RPN = 9 × 3 × 4 = **108** → Flag_High_RPN = True, Flag_High_Severity = True, Risk_Tier = Red
